Burned Area Detector Plugin

Version 1.0.0

Description

This QGIS plugin detects burned areas from Sentinel-2 satellite imagery using advanced fuzzy logic and region growing algorithms. It provides a user-friendly interface for wildfire monitoring and post-fire damage assessment.

Features

  • Automatic burned area detection from Sentinel-2 imagery
  • Fuzzy logic algorithm for accurate classification
  • Region growing segmentation to refine results
  • Cloud and water masking options
  • Configurable thresholds for fine-tuning
  • GeoTIFF output with georeferencing

Installation

  1. Open QGIS
  2. Go to Plugins → Manage and Install Plugins
  3. Search for "Burned Area Detector"
  4. Click Install Plugin

Usage

  1. Load your pre-fire and post-fire Sentinel-2 images in QGIS
  2. Open the plugin from the toolbar or menu
  3. Select the pre-fire and post-fire layers
  4. Choose an output location for the burned area map
  5. Adjust optional parameters if needed
  6. Click OK to process
  7. The result will be automatically added to your map

Input Requirements

  • Format: GeoTIFF (.tif, .tiff)
  • Sensor: Sentinel-2 Level-2A (recommended)
  • Bands: Should include Red, NIR, SWIR bands
  • Projection: Same projection for both images
  • Extent: Images should cover the same area

Parameters

  • Seed Threshold (0-1): Higher values are more conservative. Default: 0.9
  • Grow Threshold (0-1): Lower values are more aggressive. Default: 0.1
  • Mask Clouds: Remove cloud-affected pixels
  • Mask Water: Remove water bodies

Output

The plugin generates a GeoTIFF file with the burned area classification:

  • Value 1: Burned area
  • Value 0: Unburned area
  • Value 255: No data / masked pixels

Algorithm

The detection algorithm combines multiple spectral indices (NDVI, NBR) with fuzzy set theory to compute membership probabilities. A region growing approach then refines the classification, balancing commission and omission errors.
